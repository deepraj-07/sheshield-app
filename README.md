## ūüéĮ What is SheShield?

SheShield is a **production-grade women's safety application** engineered for real-world emergency scenarios. When danger strikes, SheShield responds in seconds ‚ÄĒ silently capturing evidence, alerting trusted contacts, and coordinating multi-layered emergency response without requiring the user to navigate complex menus.

> Built on Clean Architecture principles, SheShield is designed to work under pressure: every SOS step is non-blocking, failure-resilient, and independently logged.

<br/>

## ‚ú® Core Capabilities

| Feature | Description |
|---|---|
| ūüÜė **Multi-Trigger SOS** | Button hold, voice command, shake gesture, bracelet tap, audio scream detection |
| ūüďć **Real-Time GPS** | Live location tracking with reverse geocoding and location sharing links |
| ūüďĻ **Auto Evidence Capture** | 30-second video recording + SHA-256 tamper-proof hash on SOS trigger |
| ūüĒĶ **RPi 3B+ Bracelet** | Raspberry Pi 3B+ powered wearable for discreet SOS triggering and haptic feedback |
| ūüĎģ **Nearby Police** | OSM-powered live map of nearest police stations |
| ūüß≥ **Journey Mode** | Geofenced travel monitoring with check-in alerts |
| ūüßģ **Stealth Mode** | App disguised as a functional calculator |
| ūüďč **Evidence PDF** | Auto-generated tamper-proof PDF report for each SOS event |

## ūüŹóÔłŹ Architecture

SheShield follows **strict Clean Architecture** with clear separation of concerns across every layer.

```
lib/
‚Ēú‚ĒÄ‚ĒÄ core/
‚Ēā   ‚Ēú‚ĒÄ‚ĒÄ constants/         # app_colors.dart ¬∑ app_strings.dart ¬∑ app_constants.dart
‚Ēā   ‚Ēú‚ĒÄ‚ĒÄ theme/             # app_theme.dart (Material 3)
‚Ēā   ‚ĒĒ‚ĒÄ‚ĒÄ utils/             # logger.dart ¬∑ validators.dart ¬∑ extensions.dart
‚Ēā
‚Ēú‚ĒÄ‚ĒÄ models/                # UserModel ¬∑ ContactModel ¬∑ SosEventModel
‚Ēā                          # BraceletModel ¬∑ EvidenceModel
‚Ēā
‚Ēú‚ĒÄ‚ĒÄ services/              # Singleton services (GPS, BLE, SOS, SMS, Video...)
‚Ēú‚ĒÄ‚ĒÄ providers/             # ChangeNotifier state (AppState, Auth, SOS, Bracelet...)
‚Ēú‚ĒÄ‚ĒÄ screens/               # Full-page UI screens
‚ĒĒ‚ĒÄ‚ĒÄ widgets/               # Reusable, isolated UI components
```

### Design Principles

- **Zero Hardcoding** ‚ÄĒ All strings in `app_strings.dart`, all colors in `app_colors.dart`
- **Singleton Services** ‚ÄĒ GPS, Bluetooth, and SOS orchestration persist across navigation
- **Async-Safe** ‚ÄĒ Every I/O operation is wrapped in try/catch with structured logging
- **Failure-Resilient SOS** ‚ÄĒ Each of the 10 SOS steps can fail without blocking the next
- **Isolate-Heavy** ‚ÄĒ SHA-256 hashing and PDF generation run in separate Dart isolates
- **Provider Pattern** ‚ÄĒ `ChangeNotifier` for reactive, testable UI state

## ūüöÄ Quick Start

### Prerequisites

- Flutter `3.10+` / Dart `3.0+`
- Android Studio with SDK `31+`
- Firebase Project (free Spark tier is sufficient)
- Google Maps API Key

### 1. Clone & Install

```bash
git clone https://github.com/yourusername/sheshield.git
cd sheshield
flutter pub get
```

### 2. Configure Firebase

```bash
# Install the Firebase CLI
npm install -g firebase-tools

# Authenticate and auto-configure for Flutter
firebase login
flutterfire configure
```

### 3. Add Google Maps Key

Open `android/app/src/main/AndroidManifest.xml` and replace the placeholder:

```xml
<meta-data
    android:name="com.google.android.geo.API_KEY"
    android:value="YOUR_GOOGLE_MAPS_API_KEY"/>
```

### 4. Environment Variables

Create a `.env` file in the project root:

```env
FIREBASE_PROJECT_ID=sheshield-prod
GOOGLE_MAPS_API_KEY=your_key_here
OSM_OVERPASS_API=https://overpass-api.de/api/interpreter
```

### 5. Run & Build

```bash
# Development
flutter run

# Production APK (with obfuscation, split per ABI)
flutter build apk --obfuscate --split-per-abi
```

## ūüď° Bluetooth Protocol (Raspberry Pi 3B+)

SheShield communicates with the custom Raspberry Pi 3B+ wearable bracelet over classic Bluetooth Serial.

### Incoming ‚ÄĒ Bracelet ‚Üí App

| Command | Trigger |
|---|---|
| `SOS\n` | Bracelet button pressed |
| `SHAKE\n` | Shake gesture detected |
| `HR_DATA:[bpm]\n` | Heart rate reading |
| `BATTERY:[%]\n` | Battery level update |
| `STEALTH\n` | Activate stealth mode |

### Outgoing ‚ÄĒ App ‚Üí Bracelet

| Command | Action |
|---|---|
| `VIBRATE_SOS\n` | SOS vibration pattern |
| `LED_ON\n` / `LED_OFF\n` | Status LED control |
| `BUZZER_ON\n` / `BUZZER_OFF\n` | Audible alert control |
| `GET_HR\n` | Request current heart rate |
| `GET_BATTERY\n` | Request battery level |
